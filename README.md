Проект: API для управления игровой коллекцией «Steam Deck Game Library»
🎯 Цель
Разработать REST API для каталога игр, платформ и жанров с использованием Django REST Framework (DRF), базы данных Microsoft SQL Server, автодокументации Swagger, пагинации, троттлинга и JWT-авторизации.

📌 Выполненные шаги
Создание виртуального окружения и установка зависимостей:

Django, djangorestframework, drf-spectacular, djangorestframework-simplejwt, mssql-django.

Создание Django-проекта и приложения api_v1:

Настроены основные файлы settings.py и urls.py.

Проектирование моделей (в api_v1/models.py):

Platform (платформа: Steam Deck, PC и т.д.)

Genre (жанр игры)

Game (игра: название, описание, год выпуска, рейтинг, внешний ключ к Platform, ManyToMany к Genre)

Написание сериализаторов (serializers.py):

Преобразование объектов моделей в JSON и обратно, поддержка вложенных полей, read_only / write_only для связей.

Создание ViewSet'ов (views.py):

Полный CRUD для каждой модели, фильтрация по параметрам запроса, кеширование retrieve-запросов (drf-extensions).

Настройка маршрутизации (api_v1/urls.py):

Использование DefaultRouter для автоматической генерации эндпоинтов:

/api/v1/platforms/, /api/v1/genres/, /api/v1/games/.

Подключение базы данных Microsoft SQL Server:

Установлен драйвер mssql-django, настроены параметры подключения в DATABASES (Windows-аутентификация, сервер DESKTOP-E40V61B, база SteamDeckDB).

Выполнены миграции для создания таблиц.

Заполнение БД тестовыми данными через SQL-скрипт (платформы, жанры, игры, связи многие-ко-многим).

Настройка автоматической документации Swagger:

Подключён drf-spectacular, документация доступна по адресу /api/schema/swagger-ui/.

Добавление пагинации, троттлинга и JWT-авторизации:

Пагинация: 10 записей на страницу.

Троттлинг: 100 запросов/час для анонимов, 1000 – для авторизованных.

JWT: эндпоинты /api/token/ и /api/token/refresh/.

Настройка встроенной админ-панели:

Модели зарегистрированы, добавлены поиск и фильтры для удобного управления данными.

Тестирование API через браузер, Swagger UI и командную строку.

🛠️ Используемые технологии
Python 3.14

Django 5.2

Django REST Framework

Microsoft SQL Server + ODBC Driver 17

drf-spectacular (Swagger)

djangorestframework-simplejwt

Git (контроль версий)

🔗 Основные эндпоинты (локально)
GET /api/v1/games/ – список игр (с пагинацией, фильтрацией)

POST /api/v1/games/ – создание игры (требуется JWT)

GET /api/v1/platforms/ – список платформ

GET /api/v1/genres/ – список жанров

/admin/ – админ-панель

/api/schema/swagger-ui/ – интерактивная документация

📊 Результат
Разработано полностью рабочее API для игрового каталога, готовое к интеграции с фронтендом или мобильным приложением. База данных заполнена тестовыми записями, все связи корректны, документация актуальна.
